## Overall Impression

Developer-built tool that knows what it is. Playmat visual language is the right call. Single biggest problem: interaction architecture. window.prompt() breaks the entire UX contract. Errors disappear in 0.6s. Functional for one developer; would fail a new user in two minutes.

## What Is Working

1. Viewport-relative --cw card sizing — clamp formula fills available height without breakpoints.
2. HP gauge — color-coded bar with rem/max text gives critical state at a glance.
3. Playmat visual language — coherent, non-generic, reads as physical playmat not a dashboard.

## Priority Issues

**[P0] window.prompt() for retreat and attack targeting** — ActionBar.tsx chosenSlot() and retreat(). Suspends React rendering, removes all game-state context from view, accepts any string. Fix: inline slot selection using existing sel state machine. Set sel to { kind: "picking-retreat" }, highlight valid bench slots, click to complete.

**[P0] repeating-linear-gradient fabric texture on .mat::after** — absolute ban violation, added in previous session. Fix: delete .mat::after rule entirely.

**[P1] Error state invisible after 0.6s** — errglow animation is one-shot. err string persists at 12px in crowded flex row. Raw API error strings. Fix: persistent error banner below action buttons, clear on next success, translate API errors to player language.

**[P1] Attack buttons show no energy cost** — a.name (a.damage) only. Player must recall costs from memory. Fix: render a.cost as edot spans using energyColor() already used in Card.tsx.

**[P2] ActionBar visual hierarchy collapses under load** — up to 7 primary buttons simultaneously. TERMINAR TURNO weaker than card actions. Fix: demote utility actions to secondary, give TERMINAR TURNO distinct accent/red fill.

## Questions to Consider

1. When does window.prompt() become a modal? Drag-and-drop already solves energy/evolve targeting — are prompts intended as fallbacks or primary?
2. What does the player do while the bot thinks? If bot becomes async (LLM-backed), "Turno do bot…" becomes an untracked loading screen.
3. Is the hand tray part of the table? Closing the gap between hand and mat spatially might strengthen the physical-table metaphor more than decoration.
